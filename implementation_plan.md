# 명리 온톨로지 기반 데이팅 플랫폼 <연이아씨> 개발 로드맵 및 기획

본 문서는 <연이아씨> 플랫폼의 성공적인 구축을 위한 전체 개발 로드맵과, 가장 우선적으로 개발될 **핵심 사주 궁합 계산 로직**에 대한 구현 계획을 담고 있습니다.

## User Review Required

> **기술 스택 확정 요망**

## 📅 전체 개발 로드맵 (Roadmap)

<연이아씨>의 개발은 다음 4단계(Phase)로 진행됩니다.

### Phase 1: 핵심 사주 궁합 엔진 구축 (현재 단계)
*   **만세력 변환 모듈**: 생년월일시(KST) 기반 사주 원국(천간, 지지, 오행, 십성, 대운) 추출 로직 구현.
*   **하이브리드 궁합 알고리즘 구현**:
    *   일간(日干) 궁합 연산 로직 ($C_{il}$)
    *   오행 조화도(정량 분석 및 보완도) 연산 로직 ($C_{five}$)
    *   지지 관계성(합·충·형·해·파 및 신살) 연산 로직 ($C_{rel}$)
    *   바이오리듬 기반 보정치 연산 로직 ($C_{bio}$)
*   **종합 점수 산출**: 다차원 가중치 수식 ($C_{total}$) 적용 및 테스트 검증.

### Phase 2: 엔터프라이즈 API 및 백엔드 인프라 구축
*   **데이터베이스 설계**: 유저 정보, 사주 메타데이터, 멤버십(Starter, Pro, Master) 스키마 설계.
*   **보안 및 인증**: 청소년 가입 차단을 위한 본인 인증 로직 연동 및 개인정보(민감 정보) 암호화 보관/파기 프로세스 구축.
*   **매칭 API 구현**: 알고리즘 기반 자동 매칭 및 유저 필터링 API 구축.

### Phase 3: AI 에이전트 및 유료화 시스템 도입
*   **구독 시스템 연동**: 월간 구독형 '에로스(Eros)' 채널 결제 시스템 연동.
*   **콘텐츠 생성 로직**: 일일 궁합 카드, 타로/관상 스낵 콘텐츠 로직, 대운/연운 기반 캘린더 피드 생성 로직 구현.

### Phase 4: 모바일 애플리케이션 (프론트엔드) 개발
*   **UI/UX 구현**: '정서적 효율성'을 극대화한 프리미엄 디자인(오행 레이더 차트, 커스텀 원국표 등) 적용.
*   **핵심 기능 연동**: 1분 초간편 온보딩, 매칭 프로필 스와이프, 매니저 개입형 채팅방(연애의 경고등 포함) 구현.

## 🛠️ 상세 구현 계획: Phase 2 (엔터프라이즈 API 및 DB 구축)

Phase 1에서 완성된 사주 알고리즘 엔진을 기반으로, 실제 클라이언트(앱)와 통신할 수 있는 API 서버와 데이터를 보관할 데이터베이스를 세팅합니다.

### 1. 데이터베이스(DB) 및 ORM 세팅
*   초기 빠른 개발 및 검증을 위해 **SQLite** 데이터베이스를 사용하고, 추후 프로덕션(실서비스) 단계에서 **PostgreSQL**로 쉽게 마이그레이션할 수 있도록 **Prisma ORM**을 도입합니다.
*   **스키마 설계**:
    *   `User`: 기본 정보 (이름, 가입일, 멤버십 등급, 나이 등)
    *   `SajuProfile`: 생년월일시, 성별 등 알고리즘 연산에 필요한 정보
    *   `MatchHistory`: 유저 간 매칭 기록 및 궁합 점수 저장

### 2. Express API 서버 구축
*   **프레임워크**: `express` 및 `@types/express` 패키지 설치.
*   **인증 및 보안 (JWT)**: `jsonwebtoken`, `bcrypt` (비밀번호나 고유 인증키 해싱용) 도입.
*   **라우팅**:
    *   `POST /api/auth/verify`: (Mock) 본인 인증 진행 및 검증 토큰 발급 (실제 환경에서는 PortOne 등 연동)
    *   `POST /api/users/register`: 본인 인증 토큰 검증 후 신규 유저 생성, 사주 프로필 등록, JWT 발급
    *   `POST /api/auth/login`: 유저 로그인 및 JWT 발급
    *   `POST /api/match/calculate`: 두 유저의 ID를 받아 Phase 1에서 만든 궁합 엔진으로 $C_{total}$ 점수 및 보완도 반환
    *   `GET /api/match/recommend`: (JWT 인증 필요) 현재 로그인한 유저와 DB 내 다른 유저들의 궁합을 일괄 계산하여 상위 N명 추천 (유저 매칭 알고리즘)

### 3. 인증 및 매칭 API 구현 계획 (Phase 2 잔여 작업)
*   **본인 인증 기반 회원가입**: 로컬 개발 환경이므로 실제 본인 인증(KCB/NICE) 대신 '가상의 휴대폰 인증' API를 구현하여, 인증 완료 시 회원가입이 가능한 흐름 구축. 이후 JWT를 통한 인증 미들웨어 추가.
*   **유저 매칭 알고리즘 API**: 로그인한 유저의 정보를 바탕으로, 사주 궁합 점수가 특정 기준(예: 70점 이상)이거나 오행 보완도가 높은 유저를 우선적으로 정렬하여 반환하는 `/api/match/recommend` 구현.

## 🛠️ 상세 구현 계획: Phase 3 (AI 에이전트 및 유료화 시스템 도입)

본 단계에서는 사용자 경험을 향상시키고 비즈니스 모델을 구체화하는 작업을 진행합니다.

### 1. 유료화(멤버십) 시스템 연동
*   **구독 API (`POST /api/payment/subscribe`)**: 
    *   실제 PG사(PortOne 등) 연동 대신, 로컬 테스트용 모의 결제 API를 구현합니다.
    *   결제 성공 시 유저의 `tier` 필드를 `Starter`에서 `Pro` 또는 `Master`('에로스 채널' 접근 권한 등)로 업그레이드합니다.

### 2. AI 콘텐츠 생성 모듈 연동
*   **AI 프롬프트 엔지니어링 (`src/algorithm/ai.ts`)**: 
    *   유저의 사주 원국(천간, 지지, 오행 등)과 매칭 궁합 점수(Phase 1 결과물)를 바탕으로 친근하고 직관적인 '연애 조언' 및 '운세'를 생성하는 로직을 구축합니다.
    *   LLM 연동(OpenAI/Gemini) 또는 템플릿 기반 Mock 로직 중 사용자의 선택에 따라 개발합니다.

### 3. 신규 콘텐츠 라우터 구축
*   **`GET /api/content/daily-card`**: 로그인한 사용자의 사주 데이터를 읽어와, 오늘의 운세 및 추천 행동 지침을 반환합니다.
*   **`GET /api/content/calendar`**: 사용자의 대운/연운을 분석하여 이달의 주요 연애/인간관계 흐름을 요약해 반환합니다.

### 4. 검증 계획 (Verification Plan)
#### Automated Tests
*   `supertest`를 활용하여 `/api/payment/subscribe` 호출 시 `tier`가 정상적으로 변경되는지 테스트.
#### Manual Verification
*   Postman을 통해 토큰을 주입한 뒤 `/api/content/daily-card`를 호출하여, 반환되는 자연어 콘텐츠의 퀄리티와 사주 기반 맞춤형 정보가 제대로 포함되었는지 육안으로 확인.

## 🛠️ 상세 구현 계획: Phase 4 (프론트엔드 애플리케이션 개발)

본 단계에서는 유저가 직접 상호작용하는 클라이언트 화면을 구축합니다. 모바일 화면 비율에 맞춘 프리미엄 웹앱(PWA) 형태로 개발합니다.

### 1. 프론트엔드 프로젝트 초기화
*   `frontend` 폴더에 Vite + React + TypeScript 프로젝트를 생성합니다. (`npx create-vite@latest frontend --template react-ts`)
*   라우팅을 위해 `react-router-dom`을 설치합니다.
*   사주 오행 레이더 차트 등 데이터 시각화를 위해 `chart.js` 또는 `recharts`를 설치합니다.

### 2. 핵심 디자인 시스템 (Vanilla CSS)
*   **Aesthetics**: 딥 퍼플(Deep Purple)과 골드(Gold) 계열을 포인트 컬러로 사용하는 다크 모드 기반 프리미엄 디자인.
*   **UI 요소**: 글래스모피즘 기반의 카드 디자인, 부드러운 페이지 전환(Transition), 호버 및 클릭 시 마이크로 애니메이션 적용.
*   `index.css` 및 기능별 CSS 모듈을 통해 전역/지역 스타일을 체계적으로 관리합니다.

### 3. 주요 화면 (Pages & Components) 구현
1.  **온보딩 & 로그인 (`/login`, `/register`)**
    *   1분 초간편 온보딩: 전화번호 인증(Mock) 후 이름, 성별, 생년월일시 입력.
2.  **메인 대시보드 (`/`)**
    *   내 사주 원국표 및 오행 레이더 차트 시각화.
    *   오늘의 AI 운세 카드 (Phase 3 연동).
3.  **매칭 화면 (`/match`)**
    *   AI가 추천하는 상위 궁합 유저 목록 및 스와이프/리스트 형태의 프로필 카드.
    *   상대방의 오행 중 내가 부족한 기운을 얼마나 채워주는지 시각적 퍼센테이지(%)로 표현.
4.  **멤버십 & 캘린더 (`/eros`)**
    *   월간 연애 캘린더 피드 (Phase 3 연동) 및 Pro 멤버십 업그레이드 모의 결제 버튼.

### 4. 백엔드 연동 (API Integration)
*   로컬 서버(`http://localhost:3000`)와 통신하는 `api.ts` 유틸리티 작성.
*   로그인 시 발급받은 JWT를 LocalStorage에 저장하고, 이후 모든 요청의 `Authorization` 헤더에 포함.

### 5. 검증 계획 (Verification Plan)
*   로컬에서 `npm run dev`를 실행하여 브라우저의 '모바일 화면 뷰(iPhone 12/14 등)'로 접속합니다.
*   온보딩부터 AI 카드 확인, 유저 매칭까지의 전체 User Flow가 정상적으로 동작하는지, UI 애니메이션이 부드러운지 수동 검증합니다.

## 🛠️ 상세 구현 계획: Phase 1 (사주 궁합 계산 로직) - 완료

가장 먼저 진행할 **Phase 1: 핵심 사주 궁합 엔진**의 세부 구현 계획입니다. 이 모듈은 독립적인 패키지로 개발하여 향후 백엔드 API에 쉽게 연동되도록 설계합니다.

### 1. 만세력 데이터 구조화 (사주 원국 추출)
사용자의 생년월일시를 입력받아 다음의 JSON 형태로 구조화하는 유틸리티를 작성합니다.
*   **입력**: `YYYY-MM-DD HH:mm` (양력/음력 윤달 여부 포함)
*   **출력**: 천간(연/월/일/시), 지지(연/월/일/시), 오행 분포(목/화/토/금/수 %), 십성, 특수 신살(천을귀인, 공망 등).

### 2. 가중치 기반 연산 모듈 개발

#### [NEW] `src/algorithm/compatibility.ts`
두 유저의 사주 원국 데이터를 입력받아 $C_{total}$ 점수를 반환하는 코어 함수를 구현합니다.

*   `calculateIlganScore(userA, userB)`: 일간 상생/상극 판별 (가중치 30%)
*   `calculateFiveElementsHarmony(userA, userB)`: 서로의 오행 결핍(색상 비율) 보완도 정량 분석 (가중치 30%)
*   `calculateJijiRelations(userA, userB)`: 지지 합/충/형/해/파 분석 (가중치 20%)
*   `calculateBiorhythm(userA, userB)`: 생년월일 기반 신체/감성/지성 주기 보정 (가중치 20%)

#### [NEW] `src/algorithm/constants.ts`
*   천간/지지의 상생상극 테이블 및 신살(귀인, 공망) 매핑 상수 테이블 정의.
*   가중치 설정 상수 ($w_{il} = 0.3$, $w_{five} = 0.3$, $w_{rel} = 0.2$, $w_{bio} = 0.2$).

### 3. 검증 계획 (Verification Plan)

### Automated Tests
*   `npm run test`: Jest 등의 테스팅 프레임워크를 활용하여 다양한 생년월일 케이스에 대해 만세력이 정확히 추출되는지 검증합니다.
*   유명인 또는 극단적으로 잘 맞거나 안 맞는 사주 샘플 데이터를 주입하여 $C_{total}$ 종합 점수가 기획 의도대로 산출되는지 유닛 테스트를 진행합니다. 가중치 총합이 1.0이 되는지도 검증합니다.

### Manual Verification
*   터미널 스크립트를 통해 임의의 생년월일 두 개를 입력하고, 도출되는 궁합 점수와 오행 보완도 레이더 차트 수치를 직접 확인합니다.
